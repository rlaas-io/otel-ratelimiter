Impact Diagram (from telemetry to decisions)

Incoming telemetry

Logs, traces, metrics arrive from many services with bursty patterns.

->

Policy-aware filtering

Records are evaluated against RLAAS policy with configurable failure behavior.

->

Observable outcomes

Allowed, denied, shadow, and error paths are measurable in real time.

->

Operational actions

Operators adjust policy quickly and verify impact through dashboards.

Recommended adoption path

  1. Enable processor with fail_open: true first.
  2. Add policies in shadow mode and monitor drop-equivalent trends.
  3. Turn on watch_policies and admin API for live operations.
  4. Enforce selected policies and set alerts on drop/error ratios.
  5. Use max_batch_size where burst shape is unpredictable.
