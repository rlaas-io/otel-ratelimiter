Recommended Adoption Path

A safe, incremental onboarding sequence — from observation to full enforcement, with no risk at each step.

1

Enable with fail_open: true

Add the processor to your pipeline but let all traffic through. The processor is present and counting, but nothing is dropped. Zero risk.

2

Add policies in shadow_only mode

Write rate policies with action: shadow_only. Records that would be dropped are counted as shadow_total in metrics but still forwarded. Watch the numbers in Grafana.

3

Enable watch_policies and Admin API

Turn on hot reload so policy tweaks apply instantly. Use GET /stats and GET /config to verify the running state matches what you intended.

4

Flip shadow policies to enforcement

Once shadow metrics confirm the drop rate is acceptable, change action to deny and reload. Set Prometheus alerts on drop_ratio and evaluate_errors.

5

Add max_batch_size for bursty sources

Identify services with unpredictable batch sizes via received_total spikes. Set a conservative cap as a backstop even if their per-record policy is allow-all.

The whole path is reversible. You can move backward at any step — from enforce back to shadow, or from shadow back to fail-open — with a single policy edit and a hot reload.
