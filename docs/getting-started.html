Getting Started

From zero to production-safe telemetry throttling in under 10 minutes.

1. Add to Collector Build

processors:
  - gomod: github.com/rlaas-io/otel-ratelimiter v0.1.0

2. Create Policy File

[
  {
    "id": "logs-default",
    "name": "Default logs limiter",
    "signal_type": "log",
    "algorithm": "token_bucket",
    "rate": 1000,
    "burst": 2000,
    "action": "drop"
  }
]

3. Configure Processor

processors:
  ratelimiter:
    policy_file: /etc/otelcol/policies.json
    fail_open: true
    watch_policies: true
    watch_interval: 15s
    admin_addr: ":8090"

4. Wire Pipelines

service:
  pipelines:
    logs:
      receivers: [otlp]
      processors: [ratelimiter]
      exporters: [otlp]
    traces:
      receivers: [otlp]
      processors: [ratelimiter]
      exporters: [otlp]
    metrics:
      receivers: [otlp]
      processors: [ratelimiter]
      exporters: [otlp]

5. Validate Runtime

curl http://localhost:8090/health
# {"status":"ok","uptime_seconds":12}

curl http://localhost:8090/stats
If you are unsure about safe defaults, use fail_open: true and start policies in shadow mode before enforcing drops.
