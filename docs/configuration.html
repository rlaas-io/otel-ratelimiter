Configuration Reference

All fields supported by the current processor implementation.

processors:
  ratelimiter:
    policy_file: /etc/otelcol/policies.json
    # policies_inline: |
    #   [...]
    fail_open: true
    cache_ttl: 30s
    watch_policies: false
    watch_interval: 15s
    key_prefix: otel
    org_id: ""
    tenant_id: ""
    application: ""
    environment: ""
    max_batch_size: 0
    admin_addr: ""

Field Table

FieldTypeDefaultNotes
policy_filestringemptyPath to policy json file
policies_inlinestringemptyInline json policies
fail_openbooltrueAllow when evaluation errors happen
cache_ttlduration30sPolicy cache ttl inside engine
watch_policiesboolfalseEnable live policy reload from file
watch_intervalduration15sPolling fallback interval
key_prefixstringotelCounter namespace
org_idstringemptyDefault context field
tenant_idstringemptyDefault context field
applicationstringemptyDefault context field
environmentstringemptyDefault context field
max_batch_sizeint00 means unlimited
admin_addrstringemptyEnable admin HTTP API

Validation Rules

  • Exactly one of policy_file or policies_inline must be set.
  • cache_ttl, watch_interval, and max_batch_size cannot be negative.
