Monitoring and Alerting

Use both collector OTel metrics and admin endpoint metrics for complete operational visibility.

Primary metric families

  • ratelimiter_records_received_total
  • ratelimiter_records_dropped_total
  • ratelimiter_records_allowed_total
  • ratelimiter_records_denied_total
  • ratelimiter_records_shadow_total
  • ratelimiter_evaluate_errors_total
  • ratelimiter_policy_reloads_total

Prometheus scrape

scrape_configs:
  - job_name: otel-ratelimiter
    metrics_path: /metrics
    static_configs:
      - targets: ["otel-collector:8090"]

Useful PromQL

sum by (signal) (rate(ratelimiter_records_dropped_total[5m]))
sum by (signal) (rate(ratelimiter_records_allowed_total[5m]))
sum by (signal) (rate(ratelimiter_evaluate_errors_total[5m]))
sum by (signal) (increase(ratelimiter_policy_reloads_total[1h]))

Alert starters

  • Evaluate errors greater than 0 for 5m
  • Drop ratio greater than 20 percent for 10m
  • Unexpected reload spikes (>10 in 10m)
