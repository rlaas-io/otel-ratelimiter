Admin API

Enable with admin_addr for runtime visibility and operational controls.

processors:
  ratelimiter:
    policy_file: /etc/otelcol/policies.json
    admin_addr: ":8090"

Endpoints

MethodPathPurpose
GET/healthService liveness and uptime
GET/statsPer-signal counters
GET/configCurrent sanitized config
POST/reloadForce policy reload
GET/metricsPrometheus text metrics

Examples

curl http://localhost:8090/health
curl http://localhost:8090/stats
curl http://localhost:8090/config
curl -X POST http://localhost:8090/reload
curl http://localhost:8090/metrics

Sample /reload response

{
  "status": "reloaded",
  "engines": 3
}
Security recommendation: bind to loopback (127.0.0.1:8090) or expose behind an authenticated reverse proxy.
