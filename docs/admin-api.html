Admin API

Seven HTTP endpoints that give you runtime insight and live control over the rate limiter — without touching the collector process or restarting anything.

Enabling the Admin Server

Add admin_addr to your processor config to expose the API:

processors:
  ratelimiter:
    policy_file: /etc/otelcol/policies.json
    admin_addr: ":8090"
    admin_auth_token: "${env:OTEL_ADMIN_TOKEN}"
    admin_cors_allowed_origins: ["https://rlaas-io.github.io"]
    admin_tls_cert_file: /etc/otel/tls/admin.crt
    admin_tls_key_file: /etc/otel/tls/admin.key
Security: In production, enable admin_auth_token and TLS. For strict environments, set admin_tls_client_ca_file to require mTLS client certificates.

Endpoints

MethodPathPurpose
GET /health Service liveness and uptime. Returns 200 OK when the processor is running.
GET /stats Cumulative per-signal counters: received, allowed, dropped, denied, shadow, errors, reloads.
GET /config Sanitized view of all 23 active configuration fields — including org_id, tenant_id, service_expr, and all expression fields. Secrets are never exposed.
GET /config/policies Current active policy payload plus metadata such as source, count, checksum, and file modification timestamp.
POST /reload Force an immediate policy reload across all engines. Returns the number of engines refreshed.
GET /metrics Prometheus text format metrics — the same counters as /stats but in TYPE/HELP Prometheus format.
GET /ui/ Interactive browser-based Admin Console with 5 tabs: Overview, Live Stats, Configuration, Policies, Actions. Always public — no token required to load the page.

What /config Returns

Every configuration field is returned dynamically from the live running processor. This includes all context-extraction fields:

Context defaults

Static values injected into every RLAAS request when the telemetry record does not supply one.

  • org_id
  • tenant_id
  • application
  • environment
  • key_prefix

Extraction expressions

Dynamic lookup expressions that extract context from telemetry attributes at runtime.

  • service_expr
  • org_id_expr
  • tenant_id_expr
  • application_expr
  • environment_expr

Usage Examples

Open Admin Console UI

# Open in browser — no curl needed
http://localhost:8090/ui/

Check processor health

curl -s http://localhost:8090/health

Inspect live counters

curl -s http://localhost:8090/stats | jq .

See all active config (including org/service expressions)

curl -s http://localhost:8090/config | jq .

See active policy payload

curl -s http://localhost:8090/config/policies | jq .

Force a policy reload

curl -s -X POST http://localhost:8090/reload \
  -H "Authorization: Bearer $TOKEN" | jq .

Scrape Prometheus metrics

curl -s http://localhost:8090/metrics

Admin Console UI

Open http://<admin_addr>/ui/ in a browser to get a full read-only admin dashboard — no extra setup required. The page is served directly by the collector process and auto-detects the API endpoint from window.location.origin.

Overview tab

Status badge, uptime counter, total received / dropped counters, drop rate KPI, and per-signal breakdown cards with progress bars. Refreshes every 10 seconds automatically.

Live Stats tab

Full table with all 7 counters per signal (received, allowed, denied, shadow, errors, reloads, drop%). Drop percentage is color-coded: green < 5%, amber 5–20%, red > 20%.

Configuration tab

All 23 config fields in grouped sections — Policies, Context Defaults (org_id, tenant_id…), Expressions (service_expr, org_id_expr…), Behavior, Admin API. Boolean values shown as pills.

Policies tab

Active policy payload with metadata: source (file or inline), file path, policy count, SHA-256 checksum, last-modified timestamp, and dark-mode syntax-highlighted JSON with a Copy button.

Actions tab

One-click POST /reload button with loading state. API explorer for all 6 endpoints with inline response display and auto-generated curl commands you can copy to clipboard.

Auth & token

Enter your Bearer token in the sidebar — stored in sessionStorage for the browser tab only, sent automatically with every API call, cleared when the tab closes.

The /ui/ endpoint bypasses token auth intentionally — the HTML page contains no secrets and the user enters their token interactively inside it. All API calls made from the UI still require the token.

Operational Quick Checks

During an incident

Use /stats to see if drop rates are abnormally high, and /config to confirm which org_id / expressions are active — no Prometheus lag.

After a policy change

Call POST /reload then immediately check /config/policies SHA-256 checksum to verify the new policy loaded successfully.

Kubernetes readiness probe

Point your readinessProbe.httpGet at /health on port 8090 — returns non-200 if the processor is not ready.

Verify expression config

Use the Admin UI Config tab to confirm service_expr and org_id_expr values are correct after a collector restart or config update.

Metrics from /metrics are documented in detail on the Monitoring page, including PromQL examples and alert rules.
