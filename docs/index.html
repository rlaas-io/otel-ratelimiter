OpenTelemetry Rate Limiting For Real Production Traffic

A policy-driven processor for logs, traces, and metrics that controls cost and protects backends using the RLAAS engine with hot reload, safe rollout, and operational visibility.

Why this processor exists

High-cardinality services and noisy workloads can flood observability pipelines. This processor applies consistent policy decisions before export, reducing cost and preserving critical telemetry.

Control Spend

Apply per-signal and per-service limits so ingestion budgets stay predictable.

Protect Backends

Drop burst traffic early when backends are stressed, while keeping essential data flowing.

Safer Rollouts

Use shadow mode to measure impact first, then enforce without code redeploys.

Core capabilities

Policy sources

Load from policy_file or policies_inline. Mutually exclusive and validated at startup.

Hot reload

watch_policies uses fsnotify with polling fallback to refresh policy decisions immediately.

Batch safety limit

max_batch_size prevents oversized batches from overwhelming evaluation paths.

Operational control plane

Built-in admin server provides health, stats, config view, reload trigger, and Prometheus metrics.
