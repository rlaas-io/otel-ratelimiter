Why this processor exists

High-cardinality services and noisy workloads flood observability pipelines without warning. This processor applies consistent, policy-driven decisions before any data reaches an exporter — keeping costs predictable and backends protected.

Control Spend

Apply per-signal and per-service rate limits so ingestion budgets stay predictable even when a service has an incident-level spike.

Protect Backends

Drop burst traffic early — before it hits a storage backend or SaaS vendor — while keeping critical telemetry flowing with configurable fail-open behavior.

Safer Rollouts

Use shadow mode to measure exactly what a policy would drop, validate it in Grafana, then enforce — all without a code deploy or collector restart.

3 Signal types supported (logs, traces, metrics)
5 Admin API endpoints for runtime control
7 OTEL counters per signal type
0 Collector restarts needed for policy changes

Before & After at a Glance

One processor changes everything about how your telemetry pipeline handles volume, cost, and control.

⚠ Without Rate Limiter

  • All traffic passes through to exporters regardless of volume
  • A single noisy service can flood the entire pipeline
  • Ingestion costs spike unexpectedly during incidents
  • No visibility into which services generate the most data
  • Policy changes require collector redeployment

✓ With Rate Limiter

  • Per-service token bucket policies applied to every record
  • Burst traffic absorbed before it reaches any backend
  • Predictable cost with per-signal counters in Grafana
  • Shadow mode shows what would be dropped — safely
  • Live hot reload — policy changes take effect in seconds

Core capabilities

Policy sources

Load from policy_file or policies_inline. Mutually exclusive and validated at startup — no ambiguous configs.

Hot reload

watch_policies uses fsnotify with polling fallback to pick up policy file changes instantly. Or trigger manually with POST /reload.

Batch safety limit

max_batch_size drops excess records before the evaluation loop even runs — protecting CPU and memory on extreme burst shapes.

Operational control plane

Built-in admin server provides health, stats, config view, reload trigger, and Prometheus metrics — all over HTTP, no sidecar needed.

Start safely. Enable fail_open: true first and add policies in shadow_only mode. Observe the metrics, validate in Grafana, then enforce — with no restart at any step.
