Policy Sources

Load policies from a file path (policy_file) or embed them directly in collector YAML (policies_inline) — Helm and ConfigMap friendly.

Live Reload

Change a policy file on disk and the processor picks it up automatically via watch_policies. Or trigger a manual refresh with POST /reload.

Visibility Loop

Every decision emits OTEL counters and is exposed at /metrics. The /stats endpoint shows live totals without needing Prometheus.

4. Decision Logic at a Glance

The exact evaluation order for every record in every batch — readable in plain English.

for each record in batch:

  ① if len(batch) > max_batch_size:
      drop(reason = "batch_too_large")
      batch_exceeded_total++
      continue

  ② decision = policy_engine.evaluate(context)

  ③ if evaluate_error:
      if fail_open:  keep(record)      // default — safe
      else:          drop(record)      // strict mode
      evaluate_errors_total++
      continue

  ④ if decision == shadow:
      keep(record)
      shadow_total++                   // counted but not enforced

  ⑤ elif decision == allowed:
      keep(record)
      allowed_total++

  ⑥ else:   // denied
      drop(record)
      denied_total++
Start with shadow mode. Set action: shadow_only on any new policy so you can observe shadow_total trends in Grafana before committing to enforcement. Then flip to enforce — no redeploy.
